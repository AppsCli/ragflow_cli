## 一、概述

### 1.1 软件简介

RAGFlowCli移动客户端是一款基于 Flutter 框架开发的跨平台移动应用，用于连接 RAGFlow 服务端并访问其 RAG（检索增强生成）相关功能。软件支持 Android、iOS、Web、Windows、Linux、macOS 及 OpenHarmony 等多平台部署，为用户提供知识库管理、智能对话、搜索、Agent 管理、文件管理等核心能力。

### 1.2 技术栈

| 类别 | 技术/组件 |
|------|-----------|
| 开发框架 | Flutter 3.x、Dart 3.6+ |
| 状态管理 | Provider |
| 网络通信 | HTTP (http 1.2.0)、JSON |
| 本地存储 | SharedPreferences |
| 安全加密 | RSA 公钥加密（encrypt 5.0.3）|
| 多语言 | 11 种语言支持（中/英/日/韩/德/法/西/意/俄/阿/繁中）|

### 1.3 运行环境

- **客户端：** Android 5.0+、iOS 12+、Web 浏览器、Windows 10+、macOS 10.14+、Linux、OpenHarmony 10.3.8+
- **服务端：** 需部署 RAGFlow 服务端，并通过 REST API 与客户端交互

## 三、系统设计

### 3.1 架构设计

本软件采用典型的 **分层架构** 和 **MVC/MVVM 模式**：

```
┌─────────────────────────────────────────────────────────────┐
│                       Presentation Layer                     │
│  (Pages / Widgets - 登录、主页、知识库、聊天、搜索、Agent 等)  │
└─────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────┐
│                    State Management (Provider)               │
│  AuthProvider │ ThemeProvider │ LocaleProvider               │
└─────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────┐
│                      Business Logic Layer                    │
│  UserService │ KnowledgeService │ ChatService │ AgentService │
└─────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────┐
│                      Data Access Layer                       │
│  ApiClient (HTTP) │ Storage (SharedPreferences)              │
└─────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────┐
│                   RAGFlow Server (REST API)                  │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 核心流程设计

#### 3.2.1 应用启动流程

```
开始
  │
  ▼
WidgetsFlutterBinding.ensureInitialized()
  │
  ▼
ApiClient.initialize()  ← 从 Storage 加载服务器配置和 Token
  │
  ▼
runApp(MyApp)
  │
  ▼
AuthProvider 初始化，_restoreLoginState()
  │
  ├─ 有有效 Token 且校验通过 → 显示 HomePage
  │
  └─ 无 Token 或校验失败 → 显示 LoginPage
```

#### 3.2.2 用户登录流程

```
用户输入邮箱、密码
  │
  ▼
校验表单
  │
  ▼
检查服务器配置是否已设置
  │ 否 → 弹窗提示，跳转服务器配置页
  │
  │ 是
  ▼
RSA 加密密码（使用服务器公钥）
  │
  ▼
POST /v1/user/login
  │
  ├─ 成功 → 保存 User、Token 到 Storage 和 ServerConfig
  │         → 跳转 HomePage
  │
  └─ 失败 → 显示错误提示
```

#### 3.2.3 多服务器管理流程

```
用户可配置多个 RAGFlow 服务器
  │
  ▼
ServerConfig 列表（baseUrl, name, token, isActive）
  │
  ▼
切换激活服务器 → ApiClient.setBaseUrl()、ApiClient.setToken()
  │
  ▼
Storage.saveServerConfigs() 持久化
```

### 3.3 数据流设计

- **请求：** Page → Provider/Service → ApiClient → HTTP → RAGFlow Server  
- **响应：** Server → ApiClient (ApiResponse) → Service → Provider → Page (UI 更新)  
- **Token 管理：** 响应 Header 中的 `Authorization` 自动更新到 ApiClient 和 ServerConfig，401 时自动清除

### 3.4 安全设计

1. **密码传输：** 使用 RSA 公钥加密后传输，公钥可自定义或使用默认值  
2. **认证：** Bearer Token，随请求 Header 传递  
3. **Token 失效：** 401 时自动清除本地 Token，引导用户重新登录  
4. **本地存储：** SharedPreferences 存储用户信息、服务器配置、RSA 公钥

## 七、使用方法（用户手册）

### 7.1 安装与运行

```bash
# 安装依赖
flutter pub get

# 运行（根据连接设备选择）
flutter run          # 默认设备
flutter run -d web   # Web
flutter run -d android
flutter run -d ios
```

### 7.2 首次使用

1. **配置服务器地址**
   - 启动应用后进入登录页
   - 点击「设置服务器地址」或「配置服务器」
   - 输入 RAGFlow 服务器地址，如：`http://192.168.1.100:9380`
   - 支持配置多个服务器，可随时切换

2. **登录**
   - 输入邮箱和密码
   - 点击「登录」
   - 登录成功后自动进入主页

### 7.3 主要功能操作

| 功能 | 操作路径 |
|------|----------|
| 知识库 | 底部导航「知识库」→ 浏览/创建/管理知识库 |
| 聊天 | 底部导航「聊天」→ 选择对话 → 创建会话 → 发送消息 |
| 搜索 | 底部导航「搜索」→ 输入关键词搜索 |
| Agent | 底部导航「Agent」→ 查看和管理 AI Agent |
| 文件 | 底部导航「文件」→ 上传/下载/管理文件 |
| 账号 | 底部导航「账号」→ 个人信息、主题、服务器设置、退出登录 |

### 7.4 高级设置

- **RSA 公钥：** 登录页展开「RSA 公钥设置」，可自定义公钥或恢复默认
- **主题：** 账号页 → 主题设置 → 浅色/深色/跟随系统
- **多服务器：** 账号页 → 服务器配置 → 添加/删除/切换服务器

### 7.5 注意事项

- 服务器地址需为完整 URL（含 `http://` 或 `https://`）
- 确保 RAGFlow 服务端已正确部署并可访问
- 部分功能依赖服务端版本，请与服务器版本保持兼容
